## 1 — Short summary

You receive an old email text that contains a block:

begin 755 FLGPRNTR.COM
MOAP!@#PD=`:`-"I&Z_6Z'`&T"<TAP[1,,,#-(4A)7DQ1;AM.=5,:7W5_61EU
;:T1U&4=?1AY>&EAU95AU3AE)&D=:&T9O6%<D
`
end


This is a uuencoded binary (begin/end + filename) — decoding yields a small DOS .COM executable named FLGPRNTR.COM.

## 2 — Recon: decode the uuencode

Quick command-line step:

uudecode email.txt
# -> produces FLGPRNTR.COM


Now you have a 16-bit DOS .COM binary. No need to run it to understand what it does.

## 3 — Static reverse (what the .COM does)

Open FLGPRNTR.COM in a disassembler (ndisasm, radare2, Ghidra in 16-bit mode). The important routine looks like:

mov si, 0x011c       ; start of data
decode_loop:
  cmp byte [si], 0x24  ; check for '$' sentinel
  je  done
  xor byte [si], 0x2A  ; XOR with 0x2A
  inc si
  jmp decode_loop
done:
  mov dx, 0x011c       ; DS:DX -> string
  mov ah, 9
  int 21h              ; DOS print string terminated by '$'
  ret


Interpretation:

Data text begins at offset 0x011C.

Each byte is XORed with 0x2A until a $ (0x24) is encountered.

The program then uses DOS int 21h, ah=9 to print the result (string must end with $).

So the flag is stored XOR-encoded in the binary, terminated by $.

## 4 — Extract the flag (without executing)

Two safe options:

Option A — Emulate (dynamic):

Run in a DOS emulator (dosbox, dosemu, or QEMU) and run FLGPRNTR.COM. It prints the flag. This is quick and reliable.

Option B — Static extraction (preferred for forensics):

Open the binary in a hex editor or use xxd / hexdump to dump bytes starting from offset 0x011C until you see 0x24 (the $ sentinel).

For each byte b in that range compute b ^ 0x2A.

Convert resulting bytes to ASCII — that is the flag.

Example (conceptual with xxd / python):

# get bytes from offset 0x11C to the byte before 0x24
xxd -s 0x11c -g 1 -p FLGPRNTR.COM | tr -d '\n' > data.hex

from binascii import unhexlify
d = unhexlify(open('data.hex','rb').read())
res = []
for b in d:
    if b == 0x24: break         # stop at '$' sentinel
    res.append(chr(b ^ 0x2A))
print(''.join(res))


Applying this produces the decoded string.

## 5 — Result (flag)

The recovered flag is:

bctf{D1d_y0u_Us3_An_3mul4t0r_Or_d3c0mp1lEr}


Nice touch: the challenge hints at either dynamic execution (emulator) or static reverse (disassembler) as valid approaches.

## 6 — Lessons & takeaways

Recognize historical encodings: begin <mode> <name> / end → uuencode. Useful when handling vintage emails/Usenet dumps.

Simple XOR obfuscation + sentinel ($) is trivial to reverse statically — no need to execute binaries when you can reason about the code.

For safety, prefer static extraction or controlled emulation rather than running unknown binaries on a host.

Classic DOS conventions: .COM layout, int 21h service 9 prints $-terminated strings — helpful to quickly spot data/code patterns.
